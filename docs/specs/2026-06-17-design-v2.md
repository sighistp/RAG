# RAGv3 前端设计 v2

> **日期：** 2026-06-17
> **状态：** 待实施
> **范围：** 基于 v1 + 用户反馈的完整前端设计

---

## 1. 核心设计原则

1. **LLM 操作全部由用户触发** — 不自动总结、不自动生成目录，节省 token
2. **每个模块完全独立** — 文件/知识库/分析三个页面，不共享状态
3. **对话是真实对话** — 进入模块时自动创建，保存到历史记录
4. **分析不是书签** — 有聚类、总结、导出等实际分析能力

---

## 2. 整体布局

```
┌──────────────────────────────────────────────────────┐
│  [📄 文件] [📚 知识库] [📊 分析]            [👤 用户 ▼] │
├──────────────────────────────────────────────────────┤
│                                                      │
│              当前页面的完整内容                          │
│              （每个页面独立，不共享状态）                 │

```

- 顶部导航：三个模块 Tab + 用户头像下拉（设置、退出）
- 下方内容区：每个页面完全独立

### 3.2 核心逻辑

- 进入文件模块时自动创建一个新对话（mode='file'）
- 对话保存到历史记录，切换页面后回来还在
- 选择文件后检索范围限定
- 提问后可点击 [📊 添加到分析]（LLM 建议 + 用户确认）

## 4. 知识库模块

### 4.1 流程

```
进入知识库页面
  → 显示 KB 列表（无侧边栏，右侧全宽）
  → 用户选择一个 KB
  → 进入 KB 详情页（有侧边栏，显示该 KB 的对话历史）
  → 自动创建新对话（关联到该 KB）
  → 显示 KB 详情（目录 + 文档 + 聊天）
```

### 4.2 KB 列表页（无侧边栏）

### 4.4 目录和概述

- **目录：** 用户点击 [生成目录] 后 LLM 自动生成，可编辑/重新生成/删除
- **概述：** 用户点击 [生成概述] 后 LLM 自动生成，可编辑/重新生成/删除
- **文档列表：** 显示文件名、大小、状态

### 4.5 设置按钮菜单

```
[⚙️] 菜单：
├── 编辑名称
├── 生成目录（已有 = 重新生成）
├── 生成概述（已有 = 重新生成）
├── 添加内容（手动补充）
└── 删除目录/概述
```

## 5. 分析模块

### 5.1 页面结构（侧边栏显示卡片组列表）

```
┌──────────────────────────────────────────────────────┐
│  [📄 文件] [📚 知识库] [📊 分析]            [👤 用户 ▼] │
├────────────┬─────────────────────────────────────────┤
│ 卡片组     │  分析报告                [+ 新建卡片组]   │
│            │                                          │
│ 故障排查 ● │  ┌────────────────────────────────────┐ │
│ 部署相关   │  │ 📋 故障排查              [⚙️] [展开] │ │
│ 安全配置   │  │────────────────────────────────────│ │
│            │  │ 摘要：本组问题主要涉及系统故障...     │ │
│            │  │ [编辑摘要] [重新总结] [删除摘要]     │ │
│            │  │────────────────────────────────────│ │
│            │  │ 1. 服务挂了怎么恢复？      [× 删除] │ │
│            │  │ 2. 错误码 2001 什么意思？  [× 删除] │ │
│            │  │ 3. 如何查看日志？          [× 删除] │ │
│            │  │ [+ 添加问题]                        │ │
│            │  └────────────────────────────────────┘ │
│            │                                          │
│            │  ┌────────────────────────────────────┐ │
│            │  │ 📋 部署相关              [⚙️] [展开] │ │
│            │  │────────────────────────────────────│ │
│            │  │ 摘要：（未生成）                     │ │
│            │  │ [生成摘要]                          │ │
│            │  │────────────────────────────────────│ │
│            │  │ 1. Docker 怎么部署？       [× 删除] │ │
│            │  │ [+ 添加问题]                        │ │
│            │  └────────────────────────────────────┘ │
└────────────┴─────────────────────────────────────────┘
```

**侧边栏：** 显示卡片组列表，点击跳转到对应卡片。

### 5.2 问题归入卡片（LLM 建议 + 用户确认）

```
用户点击 [📊 添加到分析]
  → 弹窗打开：
      ┌─ 添加到分析 ─────────────────────┐
      │                                  │
      │  🤖 建议：放入「故障排查」（加载中）│  ← LLM 异步加载
      │                                  │
      │  ── 已有卡片 ──                   │
      │  ○ 故障排查（3 个问题）           │  ← 用户可直接选
      │  ○ 部署相关（2 个问题）           │
      │                                  │
      │  ○ + 新建卡片组                   │
      │                                  │
      │  [取消]  [确认]                   │
      └──────────────────────────────────┘
  → 用户不等 LLM 就能操作
  → LLM 建议加载完后自动高亮推荐项
  → 用户确认后保存
```

### 5.3 设置按钮菜单

```
[⚙️] 菜单：
├── 编辑名称
├── 生成摘要（已有 = 重新生成）
├── 添加内容（手动补充）
├── 删除摘要
└── 删除卡片组
```

### 5.4 分析模块功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 问题聚类 | LLM 建议放入哪个卡片，用户确认 | ✅ 本期 |
| 主题总结 | 用户触发 LLM 总结卡片主题 | ✅ 本期 |
| 导出 | 导出为 Markdown/PDF | ✅ 本期 |
| 批量问答 | 选中多个问题，一次性提问 | ❌ 推迟 |
| 趋势分析 | 按时间排序的问题列表（不做相似度聚合） | ❌ 推迟 |

## 6. 数据库变更

### 6.1 阶段 0：数据库迁移

```sql
-- conversations 表新增 mode 字段
ALTER TABLE conversations ADD COLUMN mode TEXT DEFAULT 'file';
-- 现有对话全部默认为 file 模式

-- analysis_cards 新增 summary 字段
ALTER TABLE analysis_cards ADD COLUMN summary TEXT DEFAULT '';
```

**注意：** analysis_cards 不需要 toc 字段（分析卡片是问题集合，不需要目录）。

### 6.2 迁移策略

- 现有对话全部默认标记为 file 模式
- 用户需要时手动创建新的 KB/分析对话

## 7. API 变更

| 端点 | 方法 | 说明 |
|------|------|------|
| `/analysis/cards/{id}/summary` | PUT | 更新卡片摘要 |
| `/analysis/cards/{id}/summary/generate` | POST | LLM 生成摘要 |
| `/analysis/suggest-card` | POST | LLM 建议问题放入哪个卡片 |
| `/knowledge-bases/{id}/toc/generate` | POST | LLM 生成目录 |
| `/knowledge-bases/{id}/overview/generate` | POST | LLM 生成概述 |
