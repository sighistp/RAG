# RAGv3 前端设计 v2

> **日期：** 2026-06-17
> **状态：** 待实施
> **范围：** 基于 v1 + 用户反馈的完整前端设计

---

## 1. 核心设计原则

1. **LLM 操作全部由用户触发** — 不自动总结、不自动生成目录，节省 token
2. **每个模块完全独立** — 文件/知识库/分析三个页面，不共享状态
3. **对话是真实对话** — 进入模块时自动创建，保存到历史记录
4. **分析不是书签** — 有聚类、总结、导出等实际分析能力

---

## 2. 整体布局

```
┌──────────────────────────────────────────────────────┐
│  [📄 文件] [📚 知识库] [📊 分析]            [👤 用户 ▼] │
├──────────────────────────────────────────────────────┤
│                                                      │
│              当前页面的完整内容                          │
│              （每个页面独立，不共享状态）                 │
│                                                      │
└──────────────────────────────────────────────────────┘
```

- 顶部导航：三个模块 Tab + 用户头像下拉
- 下方内容区：每个页面完全独立

### 3.2 核心逻辑

- 进入文件模块时自动创建一个新对话（mode='file'）
- 对话保存到历史记录，切换页面后回来还在
- 选择文件后检索范围限定
- 提问后可点击 [📊 添加到分析]（LLM 建议 + 用户确认）

## 4. 知识库模块

### 4.1 流程

```
进入知识库页面
  → 显示 KB 列表
  → 用户选择一个 KB
  → 自动创建新对话（关联到该 KB）
  → 显示 KB 详情（目录 + 文档 + 聊天）
```

**核心：** 先选 KB，再建对话。用户不需要手动点"新建对话"。

### 4.2 页面结构

**KB 列表页：**

```
┌──────────────────────────────────────────────────────┐
│  [📄 文件] [📚 知识库] [📊 分析]            [👤 用户 ▼] │

│            │  │ 5 个文档     │  │ 3 个文档     │       │
│            │  │ [进入] [删除]│  │ [进入] [删除]│       │
│            │  └─────────────┘  └─────────────┘       │
└────────────┴─────────────────────────────────────────┘
```

**KB 详情页（左右分栏）：**

```
┌──────────────────────────────────────────────────────┐

│            │                 │                       │
│            │  📝 概述         │  ┌──────────────┐ [→]│

└────────────┴─────────────────┴───────────────────────┘
```

### 4.3 目录和概述

- **目录：** 用户点击 [生成目录] 后 LLM 自动生成，可编辑/重新生成/删除
- **概述：** 用户点击 [生成概述] 后 LLM 自动生成，可编辑/重新生成/删除
- **文档列表：** 显示文件名、大小、状态

### 4.4 设置按钮菜单

```
[⚙️] 菜单：
├── 编辑名称
├── 生成目录（已有 = 重新生成）
├── 生成概述（已有 = 重新生成）
├── 添加内容（手动补充）
└── 删除目录/概述
```

## 5. 分析模块

### 5.1 页面结构

```
┌──────────────────────────────────────────────────────┐
│  [📄 文件] [📚 知识库] [📊 分析]            [👤 用户 ▼] │
├────────────┬─────────────────────────────────────────┤
│            │  分析报告                [+ 新建卡片组]   │
│            │                                          │
│            │  ┌────────────────────────────────────┐ │
│            │  │ 📋 故障排查              [⚙️] [展开] │ │
│            │  │────────────────────────────────────│ │
│            │  │ 摘要：本组问题主要涉及系统故障...     │ │
│            │  │ [编辑摘要] [重新总结] [删除摘要]     │ │
│            │  │────────────────────────────────────│ │
│            │  │ 1. 服务挂了怎么恢复？      [× 删除] │ │
│            │  │ 2. 错误码 2001 什么意思？  [× 删除] │ │
│            │  │ 3. 如何查看日志？          [× 删除] │ │
│            │  │ [+ 添加问题]                        │ │
│            │  └────────────────────────────────────┘ │
│            │                                          │
│            │  ┌────────────────────────────────────┐ │
│            │  │ 📋 部署相关              [⚙️] [展开] │ │
│            │  │────────────────────────────────────│ │
│            │  │ 摘要：（未生成）                     │ │
│            │  │ [生成摘要]                          │ │
│            │  │────────────────────────────────────│ │
│            │  │ 1. Docker 怎么部署？       [× 删除] │ │
│            │  │ [+ 添加问题]                        │ │
│            │  └────────────────────────────────────┘ │
└────────────┴─────────────────────────────────────────┘
```

### 5.2 问题归入卡片

```
用户在文件/KB 模块提问后点击 [📊 添加到分析]
  → LLM 分析问题主题
  → 建议放入哪个已有卡片（或建议新建）
  → 用户确认/修改
  → 问题保存到指定卡片
```

### 5.3 设置按钮菜单

```
[⚙️] 菜单：
├── 编辑名称
├── 生成摘要（已有 = 重新生成）
├── 添加内容（手动补充）
├── 删除摘要
└── 删除卡片组
```

### 5.4 分析模块增强功能

| 功能 | 说明 |
|------|------|
| 问题聚类 | LLM 建议放入哪个卡片，用户确认 |
| 主题总结 | 用户触发 LLM 总结卡片主题 |
| 批量问答 | 选中多个问题，一次性提问 |
| 导出 | 导出为 Markdown/PDF |
| 趋势分析 | 哪些问题被问得最多 |

|------|------|------|
| `/analysis/cards/{id}/summary` | PUT | 更新卡片摘要 |

## 8. 前端组件结构

```
frontend/src/
├── views/
│   ├── LoginView.vue
│   ├── MainLayout.vue        ← 顶部导航 + 内容区
│   ├── FileModeView.vue      ← 文件模式（独立页面）
│   ├── KBModeView.vue        ← 知识库模式（独立页面）
│   ├── AnalysisModeView.vue  ← 分析模式（独立页面）
│   └── KBDetailView.vue      ← 知识库详情页
├── components/
│   ├── Topbar.vue            ← 顶部导航栏
│   ├── MessageBubble.vue     ← 消息气泡 + [📊 添加到分析]
│   ├── ChatInput.vue         ← 输入框
│   ├── AnalysisCard.vue      ← 分析卡片（可折叠 + 摘要 + 问题列表）
│   ├── KBCard.vue            ← 知识库卡片（目录 + 概述 + 文档列表）
│   └── SettingsMenu.vue      ← 设置菜单组件
├── composables/
│   └── useAnalysis.ts        ← 添加到分析逻辑
└── stores/
    ├── chat.ts               ← 对话状态（按页面独立）
    ├── files.ts              ← 文件状态
    ├── knowledge.ts           ← 知识库状态
    └── analysis.ts            ← 分析状态
```
