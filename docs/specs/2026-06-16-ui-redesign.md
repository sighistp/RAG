# RAGv3 前端 UI 重新设计

> **日期：** 2026-06-16
> **状态：** 待实施
> **范围：** 基于用户使用反馈的全面 UI 重设计

---

## 1. 设计目标

将当前"对话中心"布局改为"模式中心"布局。用户进入系统后，先选择工作模式（文件问答 / 知识库问答 / 分析），每个模式有独立的交互流程和对话历史。

**核心原则：**
- 清晰的模式切换，消除检索范围歧义
- 对话历史可跨文件/知识库模式转移
- 分析模式独立，问题从其他模式手动添加
- 布局简洁，不杂乱

## 6. 跨模式对话转移

### 6.1 触发方式

右键点击侧边栏中的对话 → 出现上下文菜单。

### 6.2 菜单选项

```
右键菜单：
├── 删除
├── 转移到「文件」模式    ← 仅在知识库模式下显示
└── 转移到「知识库」模式  ← 仅在文件模式下显示
```

### 6.3 转移逻辑

- **文件 ↔ 知识库：** 可互转，对话的检索范围随之改变
- **文件/知识库 → 分析：** 不可转移，分析模式的问题是手动添加的
- **分析 → 文件/知识库：** 不可转移

### 6.4 数据库变更

对话表新增 `mode` 字段：

```sql
ALTER TABLE conversations ADD COLUMN mode TEXT DEFAULT 'file';
-- mode: 'file' / 'kb' / 'analysis'
```

转移操作 = `UPDATE conversations SET mode = ? WHERE id = ?`

## 8. 页面切换流程

```
用户进入系统
  ↓ 默认进入「文件」模式
  ↓

点击侧边栏「文件」
  ↓ 右侧显示文件管理 + 文件聊天
  ↓ 侧边栏显示文件模式的对话历史

点击侧边栏「知识库」
  ↓ 右侧显示知识库列表 + 知识库聊天
  ↓ 侧边栏显示知识库模式的对话历史

点击侧边栏「分析」
  ↓ 右侧显示问题卡片组
  ↓ 侧边栏显示分析模式的卡片组列表（或空）

切换模式时：
  - 侧边栏对话列表自动过滤
  - 右侧内容区切换
  - 不清除聊天历史（每个模式独立保存）
```

## 9. 数据库变更汇总

| 表 | 变更 | 说明 |
|----|------|------|
| conversations | 新增 mode 字段 | file / kb / analysis |
| analysis_cards | 新建 | 卡片组（id, name, mode, user_id） |
| analysis_questions | 新建 | 问题（id, card_id, question, source_message_id） |

### 新表结构

```sql
CREATE TABLE analysis_cards (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    mode TEXT DEFAULT 'file',  -- 来源模式
    user_id INTEGER,
    created_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE analysis_questions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    card_id INTEGER NOT NULL,
    question TEXT NOT NULL,
    source_message_id INTEGER,  -- 原始消息 ID（可选）
    created_at TEXT DEFAULT (datetime('now')),
    FOREIGN KEY (card_id) REFERENCES analysis_cards(id)
);
```

## 10. API 变更汇总

| 端点 | 方法 | 说明 |
|------|------|------|
| `/conversations` | POST | 新增 mode 参数 |
| `/conversations/{id}/transfer` | PUT | 转移对话到另一个模式 |
| `/analysis/cards` | GET | 获取卡片组列表 |
| `/analysis/cards` | POST | 创建卡片组 |
| `/analysis/cards/{id}` | DELETE | 删除卡片组 |
| `/analysis/cards/{id}/name` | PUT | 重命名卡片组 |
| `/analysis/cards/{id}/questions` | POST | 添加问题到卡片组 |
| `/analysis/cards/{id}/questions/{qid}` | DELETE | 删除问题 |
