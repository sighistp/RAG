## 0. 设计系统

### 设计原则

1. **目的明确** — 每个组件只有一个任务，用户一眼就知道它是干什么的。没有装饰性元素。
2. **不眼花缭乱** — 没有多余的动画、渐变、阴影。每个视觉元素都有理由存在。
3. **严谨的 AI** — 专业感、可信度。像一个工具，不像一个 demo。
4. **运行思路明确** — 组件的状态变化用户能预期：点击会发生什么，等待时看到什么，完成后看到什么。

### 视觉规范

| 维度 | 规范 |
|------|------|
| 配色 | 主色 #1E293B（深灰），辅色 #64748B，强调色 #2563EB（蓝），背景 #F8FAFC |
| 字体 | 标题 Poppins 600，正文 Open Sans 400/500，代码 JetBrains Mono |
| 圆角 | 按钮 8px，卡片 12px，输入框 8px，头像 50% |
| 阴影 | 仅卡片和弹窗有阴影，其他元素无阴影 |
| 间距 | 基础单位 8px，组件间距 16px，区块间距 24px |
| 动画 | 仅过渡效果（200ms ease），无装饰性动画 |

### 组件设计原则

| 原则 | 说明 | 示例 |
|------|------|------|
| 一个组件一件事 | 不在按钮里放图标+文字+下拉 | 发送按钮只有图标 |
| 状态可感知 | hover、active、loading、disabled 状态明确区分 | 按钮 hover 变深色，loading 显示 spinner |
| 信息层级清晰 | 标题 > 正文 > 辅助文字，用字号和颜色区分 | 模式名 16px 粗体，对话标题 14px 正常，时间 12px 灰色 |
| 交互可预期 | 点击任何元素，用户知道会发生什么 | 点对话 → 打开对话，点删除 → 弹确认框 |
| 错误有反馈 | 操作失败时告诉用户为什么，不只是红色提示 | "上传失败：文件格式不支持" 而不是 "上传失败" |

│                                                  │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐           │
│  │ 📄 新建│ │ 📄 李永红│ │ 📄 深度 │ │ ...    │  │

- 文件下方显示：文件名、大小、标签

### 3.4 检索范围

```

```

## 6. 跨模式对话转移

> **优先级：低。** 此功能推迟到后续版本，先做好三种核心模式。

### 6.1 触发方式

右键点击侧边栏中的对话 → 出现上下文菜单。

### 6.2 菜单选项

```
右键菜单：
├── 删除
├── 转移到「文件」模式    ← 仅在知识库模式下显示
└── 转移到「知识库」模式  ← 仅在文件模式下显示
```

### 6.3 转移逻辑

- **文件 ↔ 知识库：** 可互转，对话的检索范围随之改变
- **文件/知识库 → 分析：** 不可转移，分析模式的问题是手动添加的
- **分析 → 文件/知识库：** 不可转移

### 6.4 数据库变更

对话表新增 `mode` 字段：

```sql
ALTER TABLE conversations ADD COLUMN mode TEXT DEFAULT 'file';
-- mode: 'file' / 'kb' / 'analysis'
```

**迁移策略：** 现有对话全部默认标记为 file 模式。用户需要时手动转移。这在初期是可以接受的。

转移操作 = `UPDATE conversations SET mode = ? WHERE id = ?`

## 8. 页面切换流程

```
用户进入系统
  ↓ 默认进入「文件」模式
  ↓

点击侧边栏「文件」
  ↓ 右侧显示文件管理 + 文件聊天
  ↓ 侧边栏显示文件模式的对话历史

点击侧边栏「知识库」
  ↓ 右侧显示知识库列表 + 知识库聊天
  ↓ 侧边栏显示知识库模式的对话历史

点击侧边栏「分析」
  ↓ 右侧显示问题卡片组
  ↓ 侧边栏显示分析模式的卡片组列表（或空）

切换模式时：
  - 侧边栏对话列表自动过滤
  - 右侧内容区切换
  - 不清除聊天历史（每个模式独立保存）
```

## 9. 数据库变更汇总

| 表 | 变更 | 说明 |
|----|------|------|
| conversations | 新增 mode 字段 | file / kb / analysis |
| analysis_cards | 新建 | 卡片组（id, name, mode, user_id） |
| analysis_questions | 新建 | 问题（id, card_id, question, source_message_id） |

### 新表结构

```sql
CREATE TABLE analysis_cards (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    user_id INTEGER,
    created_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE analysis_questions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    card_id INTEGER NOT NULL,
    question TEXT NOT NULL,
    answer TEXT,                    -- 原始回答（可选）
    source_mode TEXT,               -- file / kb（记录来源模式）
    source_message_id INTEGER,
    created_at TEXT DEFAULT (datetime('now')),
    FOREIGN KEY (card_id) REFERENCES analysis_cards(id)
);
```

## 10. 组件运行行为指南

每个组件的行为必须让用户可预期。以下是每个核心组件的运行逻辑：

### 10.1 模式切换（侧边栏三入口）

```
用户点击「文件」
  → 侧边栏：文件模式对话列表
  → 右侧：文件管理 + 文件聊天
  → 当前模式高亮，其他模式变灰
  → 不清除已有对话数据

用户点击「知识库」
  → 侧边栏：知识库模式对话列表
  → 右侧：知识库列表 / 知识库详情
  → 当前模式高亮

用户点击「分析」
  → 侧边栏：分析卡片组列表
  → 右侧：卡片组展示 + 问题管理
```

**关键：** 切换模式不会丢失任何数据。每个模式独立保存状态。

### 10.2 对话交互流程

```
用户输入问题
  → 输入框显示 loading 状态（禁用 + spinner）
  → 用户消息立即显示在聊天区域（右对齐，蓝色气泡）
  → AI 回答逐字显示（流式）（左对齐，白色气泡）
  → 流式完成后：
      → 显示来源引用（灰色标签）
      → 显示反馈按钮（👍👎，hover 才显示）
      → 显示 [添加到分析] 按钮
      → 显示追问建议（如果有）
  → 输入框恢复可用
```

### 10.3 文件选择交互

```
用户在文件模式下点击文件卡片
  → 文件高亮（蓝色边框）
  → 输入框上方显示：🔍 搜索：文件A [×]
  → 发送消息时自动携带 doc_name 参数
  → 切换对话时恢复该对话的文件选择

用户点击 [×] 清除
  → 文件取消高亮
  → 检索范围恢复为「搜索全部文件」
```

### 10.4 知识库选择交互

```
用户点击知识库卡片「进入」
  → 右侧显示该 KB 的详情（目录、概述、文档列表）
  → 自动创建新对话，检索范围锁定为该 KB
  → 输入框上方显示：🔍 在「运维知识库」中搜索

用户点击「在此知识库中提问」
  → 跳转到对话模式
  → 自动创建对话
  → 检索范围锁定为该 KB
```

### 10.5 分析卡片交互

```
用户在文件/知识库模式下提问
  → 回答下方出现 [添加到分析] 按钮
  → 用户点击 → 弹窗：
      ○ 新建卡片组（输入名称）
      ○ 添加到已有卡片组（下拉选择）
  → 问题保存到分析模式

用户在分析模式下：
  → 点击 [+ 添加问题] → 输入框弹出 → 输入问题 → 添加到当前卡片
  → 点击卡片标题 → 可编辑名称
  → 点击 [× 删除] → 二次确认后删除
  → 点击折叠/展开 → 切换卡片内容显示
```

### 10.6 跨模式转移

```
用户右键点击侧边栏对话
  → 出现上下文菜单：
      删除
      转移到「文件」模式  ← 仅知识库模式下显示
      转移到「知识库」模式 ← 仅文件模式下显示
  → 用户点击转移
  → 对话从当前模式移除
  → 出现在目标模式的对话列表中
  → 检索范围重置为目标模式的默认值
```

### 10.7 错误状态处理

| 场景 | 表现 |
|------|------|
| 网络断开 | 输入框下方显示红色提示「网络连接失败」，禁用发送按钮 |
| API 返回 500 | 聊天区域显示红色错误卡片「服务器错误，请稍后重试」 |
| 文件上传失败 | 文件卡片显示红色边框 + 错误信息 |
| 检索无结果 | 显示「未找到相关信息」+ 建议「尝试换个问法」 |
| Token 过期 | 自动跳转登录页，不清除本地数据 |
